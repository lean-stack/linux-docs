## Fehlerbehandlung in Funktionen

Eine nĂĽtzliche Konvention: Fehlermeldungen nach `stderr` schreiben, Nutzausgaben nach `stdout`. Das macht Funktionen pipe-freundlich:

```bash

    echo "[FEHLER] $*" >&2    # nach stderr

    echo "[INFO]  $*"          # nach stdout
}

    if [[ ! -f "$DATEI" ]]; then

        return 1
    fi
    cat "$DATEI"
}
```
